# Group 9: Spending Lock

## Mission

Protect a target balance.

Warn users before their spending pushes them below a safe balance.

## Required Features

Add at least these five features to your own copy of the budget tracker.

1. Set a minimum safe balance.
2. Calculate how much the user can safely spend.
3. Warn before adding an expense that breaks the safe balance.
4. Create a daily spending limit from the safe balance.
5. Allow the user to reset the safe balance target.

## Stretch Ideas

- Add emergency expense override.
- Show remaining safe spending for the week.

## Beginner Instructions

1. First make sure the base budget tracker runs.
2. Add one feature at a time.
3. After each feature, run `python main.py`.
4. If the app breaks, undo only your last change and try again.
5. Prepare a short explanation of what each feature does.

## Presentation Tip

Do not only show code. Show the menu, add sample data, run your new feature, and explain the part of the code your group changed.
