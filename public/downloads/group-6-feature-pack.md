# Group 6: Income Planner

## Mission

Plan expected money before it arrives.

Help users compare expected income with money they actually received.

## Required Features

Add at least these five features to your own copy of the budget tracker.

1. Create expected income sources such as allowance or salary.
2. Mark expected income as received.
3. Show expected income versus actual income.
4. Warn the user when expected income has not arrived.
5. Display an income-planning summary.

## Stretch Ideas

- Add recurring income.
- Add a payment due date.

## Beginner Instructions

1. First make sure the base budget tracker runs.
2. Add one feature at a time.
3. After each feature, run `python main.py`.
4. If the app breaks, undo only your last change and try again.
5. Prepare a short explanation of what each feature does.

## Presentation Tip

Do not only show code. Show the menu, add sample data, run your new feature, and explain the part of the code your group changed.
